Run as root on Debian/Ubuntu/NixOS:

curl -k PLACEHOLDER_CA_ORIGIN/root.crt -o /usr/local/share/ca-certificates/lanbat-ca.crt
update-ca-certificates

Or on Fedora/RHEL:

curl -k PLACEHOLDER_CA_ORIGIN/root.crt -o /etc/pki/ca-trust/source/anchors/lanbat-ca.crt
update-ca-trust

On NixOS (add to configuration.nix):

security.pki.certificateFiles = [ /etc/ssl/certs/lanbat-ca.crt ];

Option 1 — GUI:

  1. Download lanbat-ca.crt above.
  2. Double-click the file → Keychain Access opens.
  3. Add to System keychain.
  4. Find the certificate, double-click → Trust → "Always Trust".

Option 2 — Terminal:

curl -k PLACEHOLDER_CA_ORIGIN/root.crt -o ~/Downloads/lanbat-ca.crt
sudo security add-trusted-cert -d -r trustRoot \
  -k /Library/Keychains/System.keychain ~/Downloads/lanbat-ca.crt

Run in PowerShell as Administrator:

Invoke-WebRequest -Uri PLACEHOLDER_CA_ORIGIN/root.crt `
  -OutFile "$env:TEMP\lanbat-ca.crt" -SkipCertificateCheck
Import-Certificate -FilePath "$env:TEMP\lanbat-ca.crt" `
  -CertStoreLocation Cert:\LocalMachine\Root

Or manually:

  1. Download lanbat-ca.crt.
  2. Right-click → Install Certificate.
  3. Store location: Local Machine.
  4. Place in: Trusted Root Certification Authorities.
  1. Scan the QR code above or download the cert via browser.
  2. Go to Settings → Security → Encryption & Credentials → Install a Certificate.
  3. Choose CA Certificate.
  4. Select the downloaded .crt file.
  5. Accept the warning.
Note: On Android, user-installed CAs are not trusted by all apps by default (Chrome works; some apps enforce system trust). This is an Android security policy — nothing we can change.
  1. Scan the QR code above → Safari opens the cert download.
  2. Tap Allow to download the profile.
  3. Go to Settings → General → VPN & Device Management.
  4. Tap the profile → Install.
  5. Go to Settings → General → About → Certificate Trust Settings.
  6. Enable full trust for Lanbat Root CA.
